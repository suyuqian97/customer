1为什么 MNC 需要中国独立实例
  • 跨境数据限制：《数据安全法》《个人信息保护法》要求敏感数据本地存储，跨境传输受严格管控
  • 本地化合规要求：政府指引要求采用本地化基础设施及经认证的云服务商
  • 地缘政治风险：全球单一实例在极端场景下面临断连风险，需具备独立运营能力
  • 中国业务体量：消费品行业中国营收占比约 20%，鞋服/零售行业可高达 74%
2Adidas 当前现状与长期方案

现状：零售、WMS、OMS 等商用配套系统正在换代，当前全部对接旧 ECC，整体换代周期 2–3 年，处于系统过渡期。全球单实例方案不在考虑范围内。

长期落地方案：RISE with SAP 中国本地实例（Local Instance）+ 两层架构（2-Tier）；ERP + BTP 一体化部署；数据二层同步回 Global。不会采用全球单实例方案。

核心议题：① RISE 在国内实际落地效果与整体实施可行性，以及落地所需改造范围；② 处于换代过渡期的周边商业系统（零售 / WMS / OMS）与 S/4HANA 的集成方案。

ECC → RISE（S/4HANA Cloud）的改造范围评估分为三个维度：数据治理、系统架构分离、业务流程影响。SAP 提供标准化的三阶段评估方法论。
1MNC ECC → RISE 改造范围评估方法（五步法）
  • 定义策略选项 Define Strategy Options

    明确三种可选架构：① 全球单一实例 ② 强耦合中国实例 ③ 松耦合 2-Tier 架构。针对 Adidas 场景，重点评估选项 ② 和 ③

  • 确定评估维度 Define Evaluation Criteria

    七大维度加权评分：企业战略契合度 / 业务灵活性 / 法律合规风险 / 地缘政治风险 / 系统可维护性 / 长期运营成本 / 迁移成本与风险

  • 定性评估 Qualitative Evaluation

    逐一分析各选项在数据安全、运营连续性、业务流程影响等方面的表现，识别关键风险点

  • 定量评估 Quantitative Evaluation

    对业务流程（L3级）逐条评估影响：典型案例中 76 个 L3 流程中 32 个受影响，涉及 109 条数据流，需额外集成 11 个全球解决方案

  • 输出建议方案 Result & Recommendation

    综合评分后输出最优架构建议，典型结论：2-Tier 松耦合架构得分最高，有效平衡合规风险与业务连续性

2三类周边系统集成说明
零售 / OMS 集成

集成场景：电商平台订单（天猫/京东/抖音/自营）→ S/4 销售订单 / AR

主要数据流：
· 订单创建与状态同步
· 库存实时查询
· 财务收款凭证生成
· 退换货逆向流程

集成方式：SAP BTP Integration Suite 标准 API + 中间件适配器

WMS 集成

集成场景：S/4 物料凭证 ↔ WMS 入库/出库指令，实现库存一致性

主要数据流：
· 采购入库通知（GR）
· 销售出库指令（GI）
· 库存盘点差异同步
· 物料主数据同步

集成方式：标准 IDOC / REST API，可使用 Edge Integration Cell 降低延迟

全球 ERP 松耦合集成

集成场景：中国 RISE 实例与全球 ECC/S/4 之间保持有限数据互通

主要数据流：
· 财务汇总报告上报
· 跨公司 PO / 内部结算
· 主数据（物料/供应商）同步
· 非敏感业务数据交换

集成方式：Global Integration Suite + Local Integration Suite 双层架构

3Clean Core 集成原则（SAP 推荐）
  • 所有与 S/4HANA 的集成通过 BTP Integration Suite 统一管理，避免直接修改 S/4 核心代码
  • 本地系统优先使用 Edge Integration Cell（中国境内部署），降低数据跨境风险
  • 中国本地集成与全球集成逻辑分层管理，互不干扰
  • 所有集成开发统一在 全球 DEV 系统中管理，本地系统仅存放本地化集成逻辑

对 Adidas 的具体建议：
建议评估现有零售 OMS 与 WMS 系统的 API 开放能力，并确认是否已与其他系统有 SAP 集成经验。SAP 可提供标准的集成内容包（Integration Content）以加速对接，减少定制开发工作量。

注意事项：如客户使用非 SAP WMS（如曼哈顿、Infor 等），需评估接口标准化程度，可能需要额外的适配开发工作。

以下为 SAP 服务过的跨国企业在中国实施 RISE with SAP + 2-Tier 架构的真实案例，涵盖快消、零售、制造等行业，与 Adidas 场景高度相关。
📦 案例一：全球食品制造企业 — 中国独立实例分离项目
行业：快消 / 食品制造｜规模：140+ 国家运营，全球单一 S/4HANA
已完成
▸ 背景与挑战
  • 全球所有业务（含中国区）运行于单一 S/4HANA 实例
  • 因地缘政治风险，需将中国区剥离为独立实例
  • 不确定改造范围，CIO 无法做决策
  • 需确保剥离后全球与中国业务连续性
▸ SAP 解决方案
  • 采用集中式 + 松耦合架构（Centralized with Loosely Coupled）
  • 使用 Clone & Delete 方法（DMLT 公司代码克隆删除）建立中国实例
  • 本地 + 全球双层 Integration Suite 管理数据流
  • Stage 1 9 个月内完成上线

关键成果：32 个受影响业务流程均制定缓解方案，109 条数据流中 62 条通过集成重建，建立了风险管理计划支持 CIO 决策，提供了全面变革管理指引

💊 案例二：全球制药企业 — 中国数据合规评估
行业：生命科学 / 制药｜规模：10 年 S/4 转型计划，20+ ERP 实例
评估阶段
▸ 背景与挑战
  • 坚持"1 全球实例 + 1 流程体系"战略
  • 强大的本地 IT / 安全 / 法务团队需独立分析
  • 需深度评估 500+ 接口的跨境数据流
  • 要保持与监管机构的持续沟通，同时制定 Plan B
▸ 评估成果
  • 分析了 22+ 数字骨干系统 / 310+ 数据对象
  • 梳理了 500+ 接口 / 6 种监管场景
  • 识别 4–5 种 S/4HANA 实例选项
  • 预计成本增加 30–70%，分离周期 6–9 个月

对 Adidas 的参考意义：该案例验证了在复杂全球模板下，中国实例分离在技术上是可行的，但需提前做详尽的数据对象分析和接口影响评估。建议 Adidas 从 Phase I 评估开始，3–8 周内完成范围摸底。
